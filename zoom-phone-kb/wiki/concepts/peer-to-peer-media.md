# Peer-to-Peer Media

## Summary

A feature that allows Zoom clients and supported desk phones to send media (audio/video) directly to each other rather than routing through Zoom's servers. This can improve call quality and reduce latency, but introduces design considerations around feature compatibility (e.g., recording, monitoring) and the need for client restarts after toggling.

## Key points

- Media flows directly between devices instead of through Zoom servers
- Available for Zoom clients and supported desk phones
- Requires Zoom Workplace app version 6.7.0 or later
- Client and device restart required for setting changes to take effect
- Feature rolling out in phases over several months
