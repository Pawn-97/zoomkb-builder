# Zoom Client ZPLS Validation

## Summary

The end-user-facing verification that ZPLS configuration has been successfully pushed to the Zoom desktop client. The client shows a Configured status upon initial setup and transitions to Registered once the client connects to ZPLS. This is the only end-user-visible aspect of ZPLS, making it critical that designers provide clear, reassuring feedback.

## Key points

- Status transitions: Configured -> Registered upon successful ZPLS connection
- Viewable in: Settings > Statistics > Phone tab
- Displays ZPLS IP address for user verification
- Recommended to test with simulated failover after validation
