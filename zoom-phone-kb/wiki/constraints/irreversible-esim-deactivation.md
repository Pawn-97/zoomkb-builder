# Irreversible eSIM Deactivation

## Summary

When an admin deactivates a user's eSIM, the action cannot be undone. This is explicitly noted in the UI. Design implication: a clear, unambiguous warning modal should precede this destructive action.

## Key points

- Deactivation is permanent and irreversible
- Design must include explicit warning before confirming
- Consider secondary confirmation pattern for destructive actions
