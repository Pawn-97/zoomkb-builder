# Authorize Zoom Phone in LTI Pro (End User)

## Summary

Each instructor or admin must individually authorize Zoom Phone the first time they launch it within LTI Pro. This OAuth-style redirection asks the user to log in and grant permissions. The matching-email requirement between Zoom and LMS accounts is a potential friction point that designers should account for with clear error messaging.

## Key points

- Launch LTI Pro and click the Phone button
- Complete OAuth-style connection dialog and log in with Zoom credentials
- Click Allow to grant permissions and see a success message
- Zoom account email must match LMS login email—mismatch blocks authorization
- User must also be signed into the Zoom desktop client for calls to route
