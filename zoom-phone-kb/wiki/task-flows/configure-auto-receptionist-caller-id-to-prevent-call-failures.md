# Configure Auto-Receptionist Caller ID to Prevent Call Failures

## Summary

Admins diagnose and fix call failures caused by auto-receptionist caller ID misconfiguration by navigating to the user's profile and selecting the appropriate outbound caller ID number. The flow includes a post-configuration testing step, which is critical for verification. Designers should consider inline validation or proactive warnings when a risky configuration is detected.

## Key points

- Path: Phone System Management > Users & Rooms > Users > [user] > Profile > Outbound Caller ID
- Admin selects the auto-receptionist number (or alternative) from the dropdown
- Save is required to apply the change
- Manual outbound call testing is the recommended verification step
- Troubleshooting references to error codes and anti-spam guidance if calls still fail
