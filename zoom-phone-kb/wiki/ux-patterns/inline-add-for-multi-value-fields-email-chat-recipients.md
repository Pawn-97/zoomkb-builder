# Inline Add for Multi-Value Fields (Email/Chat Recipients)

## Summary

Email recipients and chat channels use an inline 'Add' pattern rather than a separate configuration modal. The acknowledgement checkbox for email recipients ('I acknowledge that the email I filled in is correct') adds a deliberate confirmation step to prevent misdirected alerts.

## Key points

- Add button for appending email recipients and chat channels inline
- Email recipients require an explicit acknowledgement checkbox for correctness
- Chat channel addition opens a dialog for verification token and endpoint configuration
