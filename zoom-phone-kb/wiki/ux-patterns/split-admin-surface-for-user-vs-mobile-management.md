# Split Admin Surface for User vs. Mobile Management

## Summary

User identity management and mobile license assignment occur in two separate admin interfaces (User Management vs. eSIM Management). This split-surface pattern requires admins to understand the dependency order and navigate between contexts to complete a single user's setup.

## Key points

- User Management handles Basic license and user identity
- eSIM Management handles Mobile license, eSIM issuance, and roaming
- Sequential dependency: Basic license first, then Mobile license
- Design consideration: cross-link between surfaces to reduce friction
