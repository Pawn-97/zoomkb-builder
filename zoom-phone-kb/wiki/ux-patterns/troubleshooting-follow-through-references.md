# Troubleshooting Follow-Through References

## Summary

When a configuration change does not resolve an issue, the article directs users to additional resources (error codes, anti-spam guidance, network/firewall configuration) rather than providing inline troubleshooting. This multi-resource pattern means the UX for problem resolution is fragmented across multiple articles and requires users to self-navigate to the next step.

## Key points

- End-of-flow links to related troubleshooting articles
- No inline diagnostic steps beyond the primary configuration change
- References error codes, anti-spam guidance, and network configuration as next steps
- Users must self-identify which follow-up resource applies to their situation
