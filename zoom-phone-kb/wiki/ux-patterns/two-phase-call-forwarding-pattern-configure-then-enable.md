# Two-Phase Call Forwarding Pattern (Configure then Enable)

## Summary

Call forwarding uses a two-phase design: pre-configure mappings during planning, then manually enable during outage. This separation is necessary because forwarding rules are complex to set up and should not be rushed during an emergency. However, the enable action must be immediately accessible and support bulk operations (Select All) for rapid activation.

## Key points

- Planning phase: Add/Save forwarding mappings (complex, pre-outage)
- Emergency phase: Enable Forwarding (simple, one-click with bulk option)
- Select All for bulk enable acknowledged as design pattern for emergency speed
- Clear visual distinction between configured-but-disabled and actively-forwarding states
