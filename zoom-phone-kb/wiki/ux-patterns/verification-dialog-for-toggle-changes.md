# Verification Dialog for Toggle Changes

## Summary

When toggling call summary templates on or off, a verification dialog appears requiring the user to confirm by clicking Enable or Disable. This pattern prevents accidental toggles of impactful settings and is part of a broader confirmation pattern used throughout the Zoom web portal. Designers should maintain consistency in dialog wording and button labeling.

## Key points

- Dialog appears after toggling a setting
- Requires explicit Enable or Disable confirmation
- Provides a safeguard against accidental configuration changes
- Appears after toggling the setting, not preemptively
- Two-button pattern: Enable/Disable vs. implicit cancel
- Consistent with verification patterns in other Zoom Phone settings
